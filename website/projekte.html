Meine Cloud-Projekte: Praktische Erfahrungen mit AWS

Cloud Resume: Eine serverlose Webseite mit AWS

Mein Cloud-Resume-Projekt ist ein Beispiel für die praktische Anwendung moderner AWS-Technologien. Die Webseite wurde vollständig serverlos entwickelt und nutzt folgende AWS-Services:

  • Amazon S3: Hosting der statischen Webseite.
  • CloudFront: Bereitstellung der Webseite über ein Content Delivery Network (CDN) mit HTTPS-Unterstützung.
  • Route 53: DNS-Verwaltung und Domain-Konfiguration.
  • API Gateway & Lambda: Implementierung eines dynamischen Besucherzählers, der die Zugriffe in einer DynamoDB-Tabelle speichert.
  • GitHub Actions: Automatisierte CI/CD-Pipeline für Tests und Deployment.

Dieses Projekt hat mir gezeigt, wie man skalierbare und kosteneffiziente Cloud-Lösungen entwickelt.

View Counter: Dynamische Besucherzählung mit AWS

Der View Counter ist ein zentrales Feature meiner Cloud-basierten Webseite, das die Anzahl der Besucher in Echtzeit erfasst und anzeigt. Die Implementierung umfasst folgende Schritte und Technologien:

  • Frontend (JavaScript): Der View Counter wurde mit JavaScript implementiert, das bei jedem Seitenaufruf eine API-Anfrage an AWS API Gateway sendet.
  • Backend (AWS Lambda): Die API-Anfrage löst eine AWS Lambda-Funktion aus, die den Besucherzähler in einer DynamoDB-Tabelle speichert und aktualisiert.
  • Datenbank (DynamoDB): Die Zugriffsdaten werden in einer DynamoDB-Tabelle gespeichert, die eine schnelle und skalierbare Datenspeicherung ermöglicht.
  • API Gateway: Die Kommunikation zwischen Frontend und Backend erfolgt über AWS API Gateway, das die Anfragen verarbeitet und an die Lambda-Funktion weiterleitet.

Herausforderungen & Lösungen: Eine der größten Herausforderungen war die Implementierung einer effizienten und sicheren Kommunikation zwischen Frontend und Backend. Durch die Nutzung von API Gateway und Lambda konnte ich eine robuste und skalierbare Lösung entwickeln, die auch bei hohen Zugriffszahlen zuverlässig funktioniert.

Ergebnis: Der View Counter zeigt nicht nur die Anzahl der Besucher an, sondern demonstriert auch meine Fähigkeiten in der Integration von AWS-Services und der Entwicklung serverloser Architekturen.

Finanz-Newsletter: In Arbeit

Derzeit arbeite ich an einem Projekt zur Entwicklung einer automatisierten Finanz-News-Webseite, die täglich Marktveränderungen und anstehende Events aus verschiedenen Quellen aggregiert. Geplante Technologien und Prozesse:

  • AWS Lambda: Datenverarbeitung und Aggregation von Finanzdaten aus APIs und Web-Scraping.
  • Amazon DynamoDB: Speicherung der aggregierten Daten für die spätere Verarbeitung.
  • API Gateway: Bereitstellung der Daten für die Webseite.
  • AWS EventBridge: Automatisierung der täglichen Updates.
  • Amazon Comprehend: Sentiment-Analyse der Finanzdaten.

Dieses Projekt wird mir die Möglichkeit bieten, komplexe Datenpipelines in der Cloud zu implementieren und zu automatisieren.
