Praxisprojekte: Moderne Cloud-Infrastruktur & DevOps-Praktiken

Cloud Resume Challenge: Serverlose Webarchitektur mit CI/CD

Dieses Projekt demonstriert meine praktischen Fähigkeiten in der AWS Cloud und mit modernen Entwicklungswerkzeugen:

  • Frontend-Hosting: Statische Website in Amazon S3, bereitgestellt über CloudFront mit HTTPS.
  • CI/CD-Pipeline: Vollautomatisierte Deployments mit GitHub Actions für Test- und Produktionsumgebungen.
  • Entwicklungsumgebung: Einsatz von VS Code, Git und GitHub für effiziente Codeentwicklung und Versionierung.
  • GitHub Copilot Integration: Erste Erfahrungen mit KI-gestützter Codeentwicklung, die meine Produktivität steigert.
  • Infrastruktur-Automatisierung: Definition von Cloud-Ressourcen als Code für reproduzierbare Bereitstellungen.
  • Domain-Management: Konfiguration und Verwaltung mehrerer Umgebungen über Route 53.

Serverloser Besucherzähler mit Lambda & DynamoDB

Eine praktische Implementierung serverloser Technologien:

  • Python-Entwicklung: Programmierung von Lambda-Funktionen in Python mit Boto3 für AWS-Ressourcenzugriff.
  • Serverless Computing: Einsatz von AWS Lambda für skalierbare, ereignisgesteuerte Verarbeitung.
  • Lambda Function URLs: Direkter API-Zugriff ohne API Gateway für einfachere Architektur.
  • NoSQL-Datenbank: Nutzung von DynamoDB als skalierbare, hochverfügbare Datenbank für Besucher-Tracking.
  • JavaScript-Frontend: Asynchrone API-Aufrufe und DOM-Manipulation für dynamische Website-Elemente.
  • Fehlerbehandlung: Robuste Error-Handling-Strategien sowohl im Frontend als auch im Backend.

API Gateway & Lambda-Integration für IP-Anzeige

Ein Projekt, das die Stärken eines vollwertigen API-Managements demonstriert:

  • REST API-Design: Erstellung einer RESTful API mit AWS API Gateway.
  • Lambda-Integration: Backend-Logik in Lambda mit Python für die Verarbeitung von Request-Headern.
  • CORS-Konfiguration: Sichere domainübergreifende Anfragen für Integration mit Webseiten.
  • Datenschutz: Implementation eines Consent-Mechanismus mit JavaScript und LocalStorage.
  • Umgebungsvariablen: Konfiguration verschiedener Einstellungen für Test- und Produktionsumgebungen.
  • Sicherheitsaspekte: Absicherung der API durch Throttling und IAM-Berechtigungen.

Container-Technologien: Praktische Erfahrung mit Docker

Im Rahmen meiner Weiterbildung habe ich umfassende praktische Erfahrungen mit Docker gesammelt:

  • Multi-Container-Umgebungen: Erfolgreiches Setup einer lokalen Entwicklungsumgebung mit mehreren Docker-Containern via Docker Compose.
  • Lambda-Container: Containerisierung von Python-Lambda-Funktionen für lokales Testing.
  • Fehlerdiagnose: Lösung komplexer Docker-Konfigurationsprobleme und Container-Networking-Herausforderungen.
  • Dockerfile-Optimierung: Anwendung von Best Practices wie Multi-Stage-Builds und nicht-Root-Benutzer für sichere, effiziente Container.
  • AWS Lambda mit Containern: Integration des AWS Lambda Runtime Interface Emulator (RIE) für lokales Testing serverloser Funktionen.

Enterprise-Level Multi-Environment-Setup mit Terraform

Implementation einer professionellen Cloud-Infrastruktur mit Terraform:

  • Multi-Environment-Architektur: Strikte Trennung von Development, Staging und Production mit separaten Konfigurationen.
  • Modulare Terraform-Struktur: Wiederverwendbare Module für Lambda, DynamoDB und API Gateway.
  • S3-Remote State: Terraform-State-Management mit S3-Buckets und DynamoDB-Locking für sichere Teamarbeit.
  • API-Konfiguration: Dynamisches Laden von umgebungsspezifischen API-Endpunkten im Frontend.
  • CORS-Konfiguration: Sichere Cross-Origin-Anfragen zwischen Frontend und Lambda Function URLs.
  • Umgebungsindikatoren: Visuelle Unterscheidung der Umgebungen durch farbcodierte Banner.
