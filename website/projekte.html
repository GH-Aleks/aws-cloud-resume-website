Meine Cloud-Projekte: Praktische Erfahrungen mit AWS

Cloud Resume: Eine serverlose Webseite mit AWS

Mein Cloud-Resume-Projekt ist ein Beispiel für die praktische Anwendung moderner AWS-Technologien. Die Webseite wurde vollständig serverlos entwickelt und nutzt folgende AWS-Services:

  • Amazon S3: Hosting der statischen Webseite.
  • CloudFront: Bereitstellung der Webseite über ein Content Delivery Network (CDN) mit HTTPS-Unterstützung.
  • Route 53: DNS-Verwaltung und Domain-Konfiguration.
  • GitHub Actions: Automatisierte CI/CD-Pipeline für Tests und Deployment.

Ergebnis: Dieses Projekt zeigt meine Fähigkeit, effiziente und kosteneffektive Cloud-Lösungen zu entwickeln.

Views Counter: Besucherzähler mit AWS

Dieses Projekt implementiert einen dynamischen Besucherzähler, der die Anzahl der Seitenaufrufe in Echtzeit anzeigt. Die Implementierung umfasst folgende Technologien und Schritte:

  • Frontend (JavaScript): Der Besucherzähler wird mit JavaScript dynamisch geladen und auf der Webseite angezeigt.
  • Backend (AWS Lambda): Eine serverlose Lambda-Funktion verarbeitet die Anfragen und aktualisiert die Anzahl der Seitenaufrufe.
  • URL-basierte Funktionalität: Die Lambda-Funktion wird über eine spezielle URL aufgerufen, die direkt mit der Funktion verknüpft ist.
  • Datenbank (DynamoDB): Die Anzahl der Seitenaufrufe wird in einer DynamoDB-Tabelle gespeichert und bei jedem Aufruf aktualisiert.

Herausforderungen & Lösungen: Eine der Herausforderungen war die effiziente Speicherung und Aktualisierung der Seitenaufrufe in DynamoDB. Durch die Nutzung von serverlosen Technologien konnte eine zuverlässige und kosteneffektive Lösung entwickelt werden.

IP-Adresse: Anzeige der Besucher-IP mit AWS API Gateway

Dieses Projekt erweitert die Webseite um die Funktion, die IP-Adresse des Besuchers in Echtzeit anzuzeigen. Die Implementierung umfasst folgende Technologien und Schritte:

  • Frontend (JavaScript): Die IP-Adresse wird mit JavaScript dynamisch geladen und auf der Webseite angezeigt.
  • Backend (AWS Lambda): Eine serverlose Lambda-Funktion verarbeitet die Anfragen und liefert die IP-Adresse des Besuchers.
  • API Gateway: Die Kommunikation zwischen Frontend und Backend erfolgt über AWS API Gateway, das die Anfragen an die Lambda-Funktion weiterleitet.

Herausforderungen & Lösungen: Eine der größten Herausforderungen war die sichere und effiziente Kommunikation zwischen Frontend und Backend. Durch die Nutzung von API Gateway konnte eine robuste und effektive Lösung entwickelt werden.
